# 手动测试指令清单 - Playground 版本

## 🎯 测试目标
在百度搜索"数字浙江"，进入百度百科，提取统一社会信用代码

---

## 📝 最简指令版本（推荐）

在 Playground (http://localhost:3000) 中依次输入以下 3 条指令：

### 指令 1：搜索并进入百科
```
在百度搜索"数字浙江"，点击百度一下，然后点击搜索结果中的百度百科链接
```

### 指令 2：查找并定位信用代码
```
在页面中查找"统一社会信用代码"，如果当前视野内没有就向下滚动，直到找到为止
```

### 指令 3：提取信用代码数据
```
提取"统一社会信用代码"的值，返回 JSON 格式：{"creditCode": "信用代码", "organizationName": "组织名称", "location": "找到的位置"}
```

---

## 📝 详细指令版本（每步验证）

如果你想更细致地控制每一步，可以分成 8 个指令：

### 步骤 1：验证页面
```
检查当前页面是否是百度首页，搜索框是否可见
```

### 步骤 2：输入搜索关键词
```
在搜索框输入"数字浙江"
```

### 步骤 3：执行搜索
```
点击"百度一下"按钮
```

### 步骤 4：等待搜索结果
```
等待搜索结果页面加载完成
```

### 步骤 5：验证百科链接
```
检查搜索结果中是否有数字浙江百度百科的链接
```

### 步骤 6：打开百度百科
```
点击数字浙江百度百科链接
```

### 步骤 7：查找信用代码（带滚动）
```
在页面中查找"统一社会信用代码"字段。如果当前视野内没有，就向下滚动 500 像素，然后再次查找。重复这个过程直到找到，最多滚动 5 次
```

### 步骤 8：提取数据
```
提取"统一社会信用代码"对应的值，以及周围的相关信息（如组织名称、类型等）。返回 JSON 格式：
{
  "creditCode": "18位统一社会信用代码",
  "organizationName": "组织名称",
  "type": "组织类型",
  "foundLocation": "在页面的哪个位置找到的"
}
```

## 🎨 Playground 操作技巧

### 1️⃣ 如何输入指令
- 在 Playground 页面底部的**输入框**中输入指令
- 按 **Enter** 或点击**发送按钮**执行
- 等待 AI 执行完成后再输入下一条指令

### 2️⃣ 如何查看执行结果
- 右侧会实时显示**浏览器截图**
- 左侧显示**执行日志**和 **AI 推理过程**
- 底部显示**返回的数据**（JSON 格式）

### 3️⃣ 如果执行失败怎么办
```
# 如果 AI 找不到元素，可以更具体地描述：
点击页面中包含"百度百科"文字的链接，通常在搜索结果的第一条或第二条

# 如果滚动后还是找不到：
继续向下滚动 800 像素

# 如果提取的数据不对：
重新提取"统一社会信用代码"字段的值，注意要提取的是18位的信用代码，而不是其他数字
```

### 4️⃣ 如何保存测试结果
- Playground 会自动生成**执行记录**
- 可以点击**导出**按钮保存为 YAML 或 JSON
- 截图会自动保存在执行记录中

## 🔍 调试技巧

### 如果搜索结果不对：
```
清空搜索框，重新输入"数字浙江"，然后搜索
```

### 如果百科链接点不到：
```
点击搜索结果列表中的第一个包含"百度百科"字样的链接
```

### 如果滚动后找不到信用代码：
```
返回页面顶部，然后慢慢向下滚动，每次滚动 300 像素
```

### 如果提取的数据格式不对：
```
重新提取，要求返回标准 JSON 格式，字段名为 creditCode，值必须是18位字符串
```
